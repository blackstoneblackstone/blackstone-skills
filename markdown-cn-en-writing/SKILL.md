# Markdown 简体中文与西文混排书写

编写、润色或检查 **中英混排 Markdown** 时，应遵循以下 **格式排版** 要点（不涉及文字风格）。来源：[selfteaching/markdown-writing-with-mixed-cn-en](https://github.com/selfteaching/markdown-writing-with-mixed-cn-en)（李笑来，要点非强制规范）。

## 何时启用

- 用户撰写/修改 `.md`、README、Wiki、提案、博客草稿
- 用户明确要求「中英混排」「中文 Markdown 规范」「排版」
- 内容以中文为主且夹杂英文、数字、代码

## 常用标点（中英混排）

除 **完整英文句/段** 外，正文使用 **全角标点**：

| 名称 | 符号 | 备注 |
|------|------|------|
| 句号 | 。 | |
| 逗号 | ， | |
| 顿号 | 、 | 并列用顿号，英文并列词在中文句中也用顿号 |
| 问号/感叹号 | ？ ！ | |
| 引号 | “” ‘’ | 弯引号 |
| 冒号/分号 | ： ； | |
| 省略号 | …… | 六个点，占两个全角宽 |
| 破折号 | —— | 两个 U+2014 |
| 圆括号 | （） | |
| 书名号 | 《》 | |
| 分隔号 | ・ | 全角 Katakana Middle Dot，非半角 `·` |

**注意：**

1. 中英混排里，**单个英文词** 需引号时，用 **全角引号**；**英文句子内** 引号用半角 `'` `"`。
2. 句末括号注释：句号在括号 **外**，如：`……（参见第三章）。`
3. 中文句内并列：`Menlo、Monaco、Courier New、monospace 等等`；纯英文句并列用半角逗号。

## 空格（半角空格 ` `）

1. **中文与英文、中文与数字** 之间：各 **一个** 半角空格。例：`这是 1 个 variable 的例子`
2. **英文/数字与全角标点** 之间：**不要** 空格。例：`这是一个 variable，这是数字 100。`
3. **全角引号外侧** 要有空格。例：`所谓的 “过早引用” 就是这样令人迷惑的。`
4. 中英文并存句中，术语用 **全角引号** 括起；**引号与后接全角括号** 之间无空格。例：`这就是所谓的 “过早引用”（Forward References）`
5. **破折号 `——` 前后** 各一个半角空格。
6. **省略号 `……` 后** 宜留半角空格再接下文（若原文档要求前后皆空，与示例一致即可）。例：`他们总是这么说…… 可实际上呢？`
7. **引号、破折号、省略号之外的全角标点** 前后 **不要** 多余空格。
8. **行内代码** `` `code` `` **与两侧中文** 各一个半角空格。

## 倾斜（强调）

- 中文 **少用** `*斜体*`，排版易难看。
- 若渲染为 `<em>`，可用 CSS 改为非斜体仅变色。
- **英文词** 必须倾斜时，可用 HTML：`<i>emphasis</i>`。

## 标题

1. 仅用 `#` 表示标题。
2. GFM 中 `#数字` 会链到 Issue，故 **`#` 后必须恰好一个空格**，如：`# 一级标题`。
3. 每个文件 **仅一个一级标题**。
4. 标题 **最多到三级**（`###`）；更深建议拆文件。

## 段落

1. **不要** 段首缩进。
2. 段与段之间 **空一行**。

## 数字与单位

1. 阿拉伯数字用 **半角**。
2. 千分位用半角逗号；**7 位及以上** 建议必加千分位，如 `21,000,000`。
3. **数值范围** 用 `～`，**两侧各半角空格**：`25 ～ 29`。
4. 范围两端的 **单位/百分号都要写全**：`25% ～ 29%`、`72kg ～ 75kg`，勿写 `25 ～ 29%`。
5. **数字与单位、货币与金额** 之间 **无空格**：`75kg`、`$85`、`25%`。

## 行内特殊字符

在 Markdown 中需转义或实体时，参考 HTML 实体表（`&amp;`、`&lt;`、`` \` `` 等）。复杂符号可查 [HTML arrows 类参考](https://www.toptal.com/designers/htmlarrows/)。

## 代理执行清单（简）

编写/修改后快速自检：

- [ ] 中文主导段落是否为全角标点（英文整句除外）
- [ ] 中英、中文与数字之间是否有半角空格
- [ ] 标点与英文/数字之间是否误加空格
- [ ] 是否只有一个 `# ` 一级标题，且子标题不超过三级
- [ ] 段落间是否空行、无段首缩进
- [ ] 数值、单位、`%`、货币是否符合上文数字节规则
